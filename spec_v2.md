# spec_v2.md - AI業務棚卸しツール SaaS版 仕様書

> "診断ツール" ではなく "業務改善OSの入口" を目指す設計

## 1. サービス概要

MVP（v1）の診断機能に加え、SaaS基盤を追加し「月100万円（Pro ¥9,800×100社）」を目指す。

| レイヤー | v1（MVP） | v2（SaaS） |
|---------|----------|-----------|
| 認証 | なし | メール+パスワード / マジックリンク / Google OAuth |
| データ永続化 | なし（「保存しない」方針） | プラン別に保存（Free:3件/Pro:無制限） |
| 履歴・比較 | なし | 前回比較・改善率トラッキング |
| ダッシュボード | なし | 改善スコア + 統計 + 部門横断 |
| レポート | CSV | CSV + PDFレポート（社内共有向け） |
| テンプレート | なし | 改善テンプレートライブラリ |
| 課金 | なし | Free / Pro / Consultant（pricing.md参照） |
| チーム | なし | Pro:3名 / Consultant:20名 |

### v1仕様との矛盾解消

v1の US-05（履歴を見たい）と D-10（サーバーに永続保存されない）の矛盾を以下で解消：
- **未ログイン**：従来通りデータは保存されない（プライバシー表記そのまま）
- **ログイン済（Free）**：直近3件をDBに保存
- **ログイン済（Pro/Consultant）**：無制限にDBに保存

## 4. 改善テンプレートライブラリ 詳細

### 4.1 テンプレートの種類

「AI化できる」「IT化できる」と判定された業務に対し、具体的な改善方法を提示する。

| カテゴリ | テンプレート名 | 内容 |
|---------|-------------|------|
| AI化 | メール返信の自動化 | プロンプト例 + ツール推薦（ChatGPT/Claude） + 導入ステップ |
| AI化 | FAQ自動生成 | 過去の問い合わせからFAQを自動生成する手順 |
| AI化 | 議事録の自動作成 | 録音→文字起こし→要約の自動化フロー |
| AI化 | レポート自動生成 | データ→分析→レポート自動化のステップ |
| AI化 | 画像・書類の自動分類 | OCR + AI分類の導入手順 |
| IT化 | 請求書発行の自動化 | クラウド請求書サービスの比較 + 導入手順 |
| IT化 | 勤怠管理のシステム化 | 勤怠管理ツール比較 + 移行ステップ |
| IT化 | ワークフロー承認の電子化 | 電子承認フロー構築の手順 |
| IT化 | 顧客管理のCRM導入 | CRMツール比較 + データ移行手順 |
| IT化 | タスク管理のツール化 | プロジェクト管理ツール比較 + チーム展開手順 |

### 4.2 テンプレートの構成

各テンプレートは以下の構成：

```
1. 概要（この改善で何が変わるか）
2. 推薦ツール（3つ程度の具体名 + 料金目安）
3. 導入ステップ（5ステップ以内）
4. 想定効果（削減時間の目安）
5. 注意点・よくある失敗
```

### 4.3 表示タイミング

- 診断結果画面で「AI化」「IT化」と判定された業務の横に「改善テンプレを見る」リンク
- テンプレートライブラリ一覧ページでカテゴリ別に閲覧
- Free: 3種まで閲覧可 / Pro: 全種閲覧可 / Consultant: 全種+カスタム登録

## 5. 前回比較・改善率の仕様

### 5.1 比較対象の特定

- 同じ部署名の直前の診断を自動で比較対象とする
- 部署名が未入力の場合はユーザーの直前の診断と比較

### 5.2 改善率スコアの算出

```
改善率(%) = (今回のAI化+IT化の割合) - (前回のAI化+IT化の割合)

例：
  前回: AI化 20% + IT化 30% = 50%（自動化可能率）
  今回: AI化 35% + IT化 35% = 70%（自動化可能率）
  改善率: +20ポイント
```

### 5.3 比較画面の表示項目

| 項目 | 表示内容 |
|------|---------|
| 前回の分類比率 | 円グラフ（小） |
| 今回の分類比率 | 円グラフ（小） |
| 改善率スコア | 大きな数字 + 矢印（↑↓→） |
| 業務ごとの変化 | 前回→今回でカテゴリが変わった業務のハイライト |
| 新規追加業務 | 今回初めて入力された業務のリスト |
